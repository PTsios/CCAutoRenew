User explicitly built an auto-sync workflow (chezmoi + SOPS + Stop/SessionStart hooks) so he never has to think about git push/pull between his MacBook and Mac Mini. He values automation over tidiness.

**Why:** he works in different times on different machines and forgets to push. Manual git was his #1 friction point. He explicitly asked for "автоматический workflow" and "all 3" upgrades (auto-wip, multi-project loop, cross-machine warning).

**How to apply:**

- **Don't run `git push` proactively** in motocontent / 2gis_parser. The Stop hook does it after every Claude response. Pushing manually is harmless but noisy.
- **Don't run `git pull`** either — SessionStart hook handles it.
- **Don't suggest cleaning up wip-commits** ("wip: auto-snapshot ...") unless the user explicitly asks. They're load-bearing — proof that work didn't get lost between machines.
- **Don't propose disabling the hooks** when something goes weird. First check `~/.claude/sync_session.sh` logic, then ask the user.
- **Don't edit `~/.claude/settings.json` directly** — that file is generated by chezmoi from `~/.local/share/chezmoi/dot_claude/settings.json.tmpl`. Edit the template (or use `chezmoi edit`), then `chezmoi cd && git commit && git push`.
- **When adding tooling that touches ~/.claude/, dotfiles, or active projects** — assume it must work symmetrically on both `dev` (MacBook) and `tech` (Mac Mini) users. Use `$HOME` or `{{ .chezmoi.homeDir }}`, never hardcode `/Users/dev` or `/Users/tech`.
- **Secrets policy**: never put plaintext secrets in `~/.claude/settings.json` permissions, never in git outside of SOPS-encrypted `.env.sops`. Old leaked keys (ScraperAPI `f863b97...`, Telegram `AAEY6n...`) have been rotated and scrubbed — don't reintroduce.
